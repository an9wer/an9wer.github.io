博客搭建小记

首先，今天日子不错（10.24），就把今天定为是这个博客的 birthday。

这个博客的目的是记录一下平日的一些感想、笔记等。主要做个备忘，以后翻翻当初写下 的东西，应该还挺有意思的。

好了，讲一下今天的主题，这个博客是如何搭建的?

其实很早就有这个想法，之前也做过实践。当时的构思是后台用 bottle，前台用 vue，数 据库用 sqlite，并且数据库中所有数据做加密处理，因为数据库本身也不大，这样可以将 整个数据库连同项目一起 commit 到 github 上，并且定时备份。照这个思路写了一段时 间，实现了数据加密入库，解密出库，以及一个简单的 index.html，页面滚动到最底部时 ，自动加载下一页的数据。后来项目就太监了。主要是因为觉得前端不是我的兴趣所在， js 特效和 css 样式的组合花样繁多，知识点太零碎，每次有问题 google 一下基本可以 解决，但是不成体系。而且整这么花里胡哨干嘛呢，字符界面才是我的 style 啊。

后来项目就一直搁置了，但是最近这个的想法又冒了出来，因为平时总会瞎折腾，总想着 能不能把这些东西都记录下来。这次我决定找个现成的方案，怎么简单怎么来。说实话， 平时瞎看一些杂七杂八的东西也是挺有好处的，之前看 https://github.com/alebcay/awesome-shellawesome shell 的时候看过过一个叫 bashblog|bashblog 的项目，当初只是 简单看了一下介绍，有那么点印象。于是这次找到这个项目的主页，仔细看了一下介绍， 并把项目下载下来，跑了一遍，还真是可以，而且 blog 风格也比较简洁，是我的 style 。关键只用 bash 命令就可以搞定了，非常 geek 。

bashblog 这个项目与我之前的思路不同，它搭建的是一个静态博客，包含 index 页面， 标签页面，并且集成了 disqus， google analytics 等功能。这样我就不需要写任何前后 台代码，只要一个 http server 就可以部署运行。

这么 geek 的项目，我当然要好好研究一下，然后重写一遍，发扬我抄轮子的精神。也就 是在研究其代码的过程中，遇到了一个非常纠结的问题，用哪个 markdown parser 比较好 ？ google 一番，看到了 这篇文章， 原来各家的 markdown parser 都有一些差异（也许 CommonMark 才是最终的解决方案）。 所以我决定应该用个标准统一的标记语言，而且语法最好也能相对简洁一点，我想到了 vimwiki，最重要的是我之前的一些笔记也是 用 vimwiki 记录的，这样子兼容性也比较好。之后我又突然意识到 vimwiki 的 :Vimwiki2HTML 命令可以直接将 vimwiki 格式的文件直接转化成 html，完全可以替代 bashblog 这个工具。没想到折腾了半天最终又回到了我一直在用的 vimwiki 。之前怎么 没发现 vimwiki 还有这功能呢。

以上，虽然折腾了半天好像什么都没做，但这个简陋的博客不是搭建起来了吗。
